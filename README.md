## MikroTik RouterOS Hotspot Pages

## 📁 هيكل الملفات

```
hotspot/
├── login.html      ← الصفحة الرئيسية (الفيديو + النموذج)
├── alogin.html     ← نجاح تسجيل الدخول
├── status.html     ← حالة الاتصال (وقت، بيانات، فصل)
├── logout.html     ← تأكيد تسجيل الخروج
├── redirect.html   ← إعادة التوجيه التلقائية
├── flogin.html     ← خطأ تسجيل الدخول (بعض المتصفحات)
├── error.html      ← أخطاء عامة للسيرفر
├── radvert.html    ← صفحة الإعلانات قبل التوجيه
├── rstatus.html    ← حالة RADIUS المتقدمة
├── style.css       ← التصميم الموحد (كل الصفحات)
├── md5.js          ← تشفير كلمة المرور (RFC 1321)
├── custom.js       ← الأنيميشن + الحماية + الفيديو
└── bg_animation.mp4  ← ← ← ارفع فيديوك هنا (أنت)
```

---

## 🚀 طريقة الرفع على MikroTik

### الطريقة 1: عبر Winbox
1. افتح **Files** في Winbox
2. انتقل إلى مجلد `hotspot/`
3. ارفع جميع الملفات داخله

### الطريقة 2: عبر FTP
```
Host: [IP السيرفر]
User: admin
Pass: [كلمة مرورك]
Path: /hotspot/
```

### الطريقة 3: عبر SCP
```bash
scp -r ./hotspot/* admin@192.168.88.1:/hotspot/
```

## 🎥 إعداد الفيديو

1. ضع ملف الفيديو في نفس مجلد الـ hotspot باسم `bg_animation.mp4`
2. افتح `custom.js` وعدّل هذه القيم:

```javascript
var CONFIG = {
  FORM_SHOW_DELAY : 3500,  // ← وقت ظهور النموذج بالميلي ثانية
                            //   3500 = 3.5 ثانية بعد بدء الفيديو
  VIDEO_DURATION  : 10000, // ← مدة الفيديو الكاملة بالميلي ثانية
};
```

### توصيات الفيديو للأداء الأمثل:
| الخاصية | القيمة المقترحة |
|---------|----------------|
| الصيغة | MP4 (H.264) |
| الدقة | 720p (1280×720) |
| معدل البت | 1–2 Mbps |
| الحجم | < 8 MB |
| الصوت | بدون صوت (muted) |

## ⚙️ إعداد MikroTik RouterOS

### في IP > Hotspot > Server Profiles:
- **Login By**: `HTTP PAP` أو `HTTP CHAP`
- **HTML Directory**: `hotspot`
- **Use RADIUS**: حسب إعدادك

### في IP > Hotspot > Walled Garden:
لا تحتاج إضافات لأن الملفات محلية.

## 🔐 ميزات الحماية المدمجة

| الميزة | الوصف |
|--------|-------|
| Anti-Clickjacking | منع تضمين الصفحة في iframe |
| XSS Protection | تعقيم جميع مدخلات المستخدم |
| Brute Force Limit | 5 محاولات / دقيقة ثم إقفال دقيقتين |
| Double Submit | منع الإرسال المزدوج |
| DevTools Block | تصعيب فتح أدوات المطور |
| CSP Headers | يُنصح بإضافتها من الـ MikroTik |
| MD5 Hashing | تشفير كلمة المرور قبل الإرسال |

## 🧪 اختبار محلي

افتح `login.html` مباشرة في المتصفح.
ستظهر الصفحة بعد انتهاء مؤقت الأنيميشن.

> **ملاحظة:** المتغيرات مثل `$(username)$` ستظهر فارغة في الاختبار المحلي — هذا طبيعي، MikroTik يعبئها تلقائياً.
